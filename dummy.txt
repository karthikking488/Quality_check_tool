# Quality_check_tool – app.py Documentation

## Overview

This Flask application provides a web-based tool for connecting to Snowflake using SSO, running SQL queries, generating and managing data quality unit tests (leveraging Snowflake Cortex AI), and exploring database metadata. The app is designed for data engineers and analysts to ensure data quality and integrity in Snowflake environments.

## Key Features

- SSO authentication with Snowflake (external browser)
- Persistent Snowflake connection management
- SQL query execution and result display
- Metadata exploration (databases, schemas, tables, views, procedures, functions)
- AI-powered unit test generation for tables, views, and procedures
- Test case storage, retrieval, and execution
- Role management and Cortex AI access checks
- Web UI with multiple pages (home, query tool, AI assistant, unit test generator)

## Main Components

### 1. Connection Management

- `create_snowflake_connection()`: Establishes a Snowflake connection using SSO and environment variables.
- `get_connection()`: Returns the global connection, reconnecting if needed.
- `close_connection()`: Closes the global connection on app shutdown.

### 2. Query Execution

- `execute_query(query)`: Executes a SQL query using the persistent connection and returns results as dictionaries.

### 3. Metadata Extraction

- `extract_referenced_objects(procedure_definition, schema)`: Parses SQL to find referenced tables/views.
- `get_object_quick_metadata(full_schema, object_name, object_type)`: Fetches columns, row count, and sample data for a table/view.

### 4. AI Prompt Generation

- `create_test_generation_prompt(...)`: Builds a detailed prompt for Cortex AI to generate test cases.
- `create_procedure_function_prompt(...)`: Specialized prompt for procedures/functions.

### 5. Test Case Handling

- `parse_test_cases(...)`: Parses AI-generated test cases from text.
- `compare_results(...)`: Compares actual query results to expected test outcomes.

## Flask Endpoints

### UI Pages

- `/` – Home page (SSO login, role selection)
- `/query-tool` – Main query tool interface
- `/ai-query` – AI query assistant
- `/unit-test-generator` – Unit test generator

### API Endpoints

- `/api/login-sso` (POST): Initiate SSO login and connect to Snowflake.
- `/api/check-connection` (GET): Check if a Snowflake connection is active.
- `/api/roles` (GET): List available roles for the user.
- `/api/set-role` (POST): Switch to a different Snowflake role.
- `/api/check-cortex` (GET): Check if the current role has Cortex AI access.
- `/api/save-tests` (POST): Save generated test cases to Snowflake.
- `/api/fetch-tests` (POST): Fetch test cases for a specific object.
- `/api/delete-test` (POST): Delete a test case.
- `/execute` (POST): Execute a SQL query from the frontend.
- `/api/databases` (GET): List all databases in the account.
- `/api/schemas` (POST): List all schemas in a database.
- `/api/objects` (POST): List objects (tables, views, etc.) in a schema.
- `/api/object-metadata` (POST): Get metadata for a specific object.
- `/api/generate-tests` (POST): Generate unit tests using Cortex AI.
- `/api/run-test` (POST): Execute a specific test case.
- `/api/table-metadata` (POST): Get metadata for a table.
- `/api/schema-metadata` (POST): Get metadata for an entire schema.
- `/api/generate-sql-from-question` (POST): Use AI to convert a natural language question into SQL.

## Usage

1. Start the app: `python app.py`
2. Open `http://localhost:5000` in your browser.
3. Sign in with SSO from the home page.
4. Use the web UI to run queries, explore metadata, and generate/manage tests.

## Environment Variables

Set the following in your `.env` file:

- `SNOWFLAKE_ACCOUNT`
- `SNOWFLAKE_USER`
- `SNOWFLAKE_WAREHOUSE`
- `SNOWFLAKE_DATABASE` (optional)
- `SNOWFLAKE_SCHEMA` (optional)
- `SNOWFLAKE_ROLE`
